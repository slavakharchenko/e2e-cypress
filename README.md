# Cypress E2E Test Template

Cypress E2E testing template using TypeScript, Page Object Model (POM), and reusable Page Element components. Separates selectors, actions, and assertions into composable layers.

## Getting Started

```bash
npm install
cp .env.example .env
```

## Commands

| Command                | Description              |
| ---------------------- | ------------------------ |
| `npm run cy:open`      | Open Cypress Test Runner |
| `npm run cy:run`       | Run tests headlessly     |
| `npm run lint`         | Check ESLint             |
| `npm run lint:fix`     | Auto-fix ESLint issues   |
| `npm run format`       | Format with Prettier     |
| `npm run format:check` | Check formatting         |

## Architecture

```
cypress/
├── api/                  # API client classes (BaseApi subclasses)
├── db/                   # Database client (Knex.js) and task functions
├── e2e/                  # Test specs (*.cy.ts)
├── page-elements/        # Reusable UI element classes
├── page-objects/         # Page Object Model classes
├── data/                 # Test data generators (Faker)
├── support/              # Custom commands & setup
└── test-plans/           # Test plan documents (generated by plan-tests agent)
```

### Three-Layer Pattern

1. **Page Elements** (`cypress/page-elements/`) — low-level, reusable UI component wrappers
2. **Page Objects** (`cypress/page-objects/`) — page-specific classes composing elements and exposing actions
3. **Test Specs** (`cypress/e2e/`) — test files using page objects, no selectors or direct DOM manipulation

## Code Style

- Double quotes, semicolons, trailing commas (all), 100 print width, 2-space indent

## Key Conventions

- **Selectors**: prefer `data-testid` > `id` > CSS class. Keep selectors in Page Element constructors only.
- **Page Elements**: extend `BaseElement`, return `this` from all methods, use `el` getter.
- **Page Objects**: extend `BasePage`, define `url`, compose elements as `readonly` properties, return `this`.
- **Test Specs**: instantiate page objects at `describe` scope, use `beforeEach` for setup, no `cy.get()` or selectors directly.
- **Test Data**: use `@faker-js/faker` in `cypress/data/`, export factory functions.
- **Assertions**: use `.should()` (auto-retry), avoid `cy.wait(ms)`.

## Anti-Patterns

- No `cy.wait(N)`, no selectors in tests, no `cy.get()` in tests, no brittle selectors, no shared mutable state, no conditional logic in tests, no assertions in page objects beyond element-level.

## Database

PostgreSQL integration via `cy.task()`. Set `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME` in `.env`. Define task functions in `cypress/db/tasks.ts`, register them in `cypress.config.ts` `setupNodeEvents`. Connection is lazy-initialized and destroyed after all tests.

## Environment

Set `ENV=dev|staging|prod` in `.env` to switch target environment. Base URLs are configured in `config.ts`.
